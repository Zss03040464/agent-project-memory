# Recovery Workflow

Use this workflow when a project appears missing or a path no longer exists.

## Principle

A missing path is evidence of a missing path, not evidence of a deleted project.

## Workflow

1. Read `project_memory/INDEX.md`.
2. Find the matching project or issue entry.
3. Read the linked record under `project_memory/records/`.
4. Check known historical paths.
5. Search the workspace root for similar names.
6. Check synchronized folders.
7. Check the recorded Git remote.
8. Read `project_memory/CLOUD.md`.
9. Check cloud mirror records.
10. Check sanitized cloud archives.
11. Report findings.
12. Only then create a replacement, and only with explicit intent.

## Unsafe shortcuts

Avoid these actions:

- assuming a project is gone because one path failed
- creating a new empty folder with the same name
- overwriting an existing folder before inspecting it
- deleting partial memory records
- uploading unfiltered archives
- storing secrets in records
