# Recovery Checklist

Use this checklist when a project path, record, or local workspace appears to be missing.

## Rule

A missing local path is not proof that a project is gone.

## Steps

1. Read `project_memory/INDEX.md`.
2. Locate the linked record under `project_memory/records/`.
3. Check all known paths listed in the record.
4. Search nearby workspace roots for similar names.
5. Check synchronized folders.
6. Check recorded Git remotes.
7. Read `project_memory/CLOUD.md`.
8. Check cloud mirror files.
9. Check sanitized archives.
10. Report what was found and what remains missing.

## Prohibited actions before recovery checks

- Do not create a new empty project with the same name.
- Do not overwrite a directory that may contain the original project.
- Do not delete partial records.
- Do not upload unfiltered archives to cloud storage.
- Do not include secrets or credentials in memory files.
